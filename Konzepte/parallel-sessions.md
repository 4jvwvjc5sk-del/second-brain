# Parallel-Sessions

#konzept #claude-code #workflow

Mehrere Claude Code Fenster gleichzeitig für verschiedene Aufgaben nutzen.

## Wie es funktioniert

- Jedes Fenster = eigene Session = eigene Aufgabe
- Alle Sessions lesen dasselbe [[second-brain-system]]
- Keine gemeinsame Chat-History → kein Token-Aufblähen

## Typische Szenarien (Remy)

- Nachts laufen lassen → Ergebnisse morgens prüfen
- Während Meeting läuft autonome Session
- Zweites Fenster für anderes Projekt

## Zweiter Account

Wenn Limit erreicht → ausloggen → zweiter Account → Second Brain Pfad angeben → sofort weiter.
